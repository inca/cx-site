## Syntax Cheatsheet

### Phrase Emphasis

    _italic_   *bold*                         {.no-highlight}

### Links

Inline:

    An [example](http://url.com/ "Title")     {.no-highlight}

Reference-style labels (titles are optional):

    An [example][id]. Then, anywhere          {.no-highlight}
    else in the doc, define the link:

      [id]: http://example.com/  "Title"

### Headers

Setext-style:

    Header 1                                  {.no-highlight}
    ========

    Header 2
    --------

atx-style:

    # Header 1                                {.no-highlight}

    ## Header 2

    ###### Header 6

with optional `id` attribute:

    # My title     {#my-title}                {.no-highlight}

    and then use it with [link](#my-title)

### Lists

Ordered, without paragraphs:

    1.  Foo                                   {.no-highlight}
    2.  Bar

Unordered, with paragraphs:

    *   A list item.                          {.no-highlight}

        With multiple paragraphs.

    *   Bar

You can nest them:

    *   Abacus                                {.no-highlight}
    
        * answer
    *   Bubbles
    
        1.  bunk
        2.  bupkis
        
            * BELITTLER
        3. burper
    *   Cunning

### Blockquotes

    > Email-style angle brackets              {.no-highlight}
    > are used for blockquotes.

    > > And, they can be nested.

    > #### Headers in blockquotes
    >
    > * You can quote a list.
    > * Etc.


### Code Spans

    `<code>` spans are delimited              {.no-highlight}
    by backticks.

    You can include literal backticks
    like `` `this` ``.


### Preformatted Code Blocks

Indent every line of a code block by at least 4 spaces or 1 tab.

    This is a normal paragraph.               {.no-highlight}

        This is a preformatted
        code block.


### Horizontal Rules

Three or more dashes or asterisks:

    ---                                       {.no-highlight}

### Manual Line Breaks

End a line with two or more spaces:

    Roses are red,                            {.no-highlight}
    Violets are blue.

### Inline HTML

The content of inline HTML is not processed,
unless wrapped into special HTML comment:

    <div>                                     {.no-highlight}
      This text remains `as is`.
      <!--#md-->
      This text *is* processed.
      <!--~-->
      Here comes unprocessed text again.
    </div>

### SmartyPants

    "quotes around text"  ->  &ldquo;curly ones&rdquo;  {.no-highlight}
    (c) and (C)           ->  &copy;
    (r) and (R)           ->  &reg;
    (tm) and (TM)         ->  &trade;
    --                    ->  &mdash;
    ...                   ->  &hellip;
    <-                    ->  &larr;
    ->                    ->  &rarr;

Ampersands in text are wrapped with `<span class="amp">`
for enhanced styling.

### Tables

    --------------------------------          {.no-highlight}
    Col1      | Col2      | Col3
    --------------------------------
    Cell 1 1  | Cell 1 2  | Cell 1 3
    Cell 2 1  | Cell 2 2  | Cell 2 3
    --------------------------------
